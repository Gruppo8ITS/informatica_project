First, we produced a very general query, and we found all the Musical Instruments present in ArCo, under the class ‘Cultural Property’.

With the following query we asked SPARQL to “Retrieve all the cultural properties present in ArCo with the name ‘musical instrument’, ordered by their label. For each of them, give me the depiction if any is available”:

PREFIX rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns

PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

PREFIX arco: https://w3id.org/arco/ontology/arco/

SELECT DISTINCT ?cp ?label ?depiction

WHERE

{?cp rdfs:label ?label ;

rdf:type arco:CulturalProperty .

FILTER(REGEX(?label , "strumento musicale" , "i"))

OPTIONAL {?cp foaf:depiction ?depiction}

}

ORDER BY (?label)

RESULTS: all the Musical Instruments present in ArCo.

Since we wanted to know the exact number of musical instruments, we added the key word COUNT to our query. We noticed that the results retrieved both from the first and second query included the language tag @eng, so we also added a FILTER to retrieve the results just once, under the tag @it. Therefore, we ran the following query:

PREFIX rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns

PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

PREFIX arco: https://w3id.org/arco/ontology/arco/

SELECT (COUNT (DISTINCT ?cp) ?as ?count)

WHERE

{?cp rdfs:label ?label ;

rdf:type arco:CulturalProperty .

FILTER(REGEX(?label , "strumento musicale" , "i"))

FILTER (LANG (?label)="it")

}

RESULTS: The results showed that there are exactly 556 musical instruments in ArCo.

As many of them were excluded from our query, we decided to explore the results of the first query, showing all the musical instruments, and we noticed that many cultural properties were not associated with the property a-dd:hasMaterial, but to a-dd:hasMaterialOrTechnique, which also specifies the material the musical instrument is made of. Therefore, we decided to also include this other property in the query, using the keyword UNION:

PREFIX rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns

PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

PREFIX arco: https://w3id.org/arco/ontology/arco/

PREFIX arco-dd: https://w3id.org/arco/ontology/denotative-description/

SELECT DISTINCT ?cp ?label ?Material ?MaterialOrTechnique

WHERE

{?cp rdfs:label ?label ;

rdf:type arco:CulturalProperty .

OPTIONAL {

{?cp a-dd:hasMaterial ?Material}

UNION

{?cp a-dd:hasMaterialOrTechnique ?MaterialOrTechnique}}

FILTER(REGEX(?label , "strumento musicale" , "i"))

FILTER (LANG (?label)="it")

}

ORDER BY (?label)

RESULTS:

With this query we did not just discover that there were instruments not associated with neither of these properties; we also observed that, notwithstanding the keyword DISTINCT in the SELECT- clause, there were duplicates in the results retrieved. However, examining them carefully, we could notice that the duplicates were simply related to the fact that a single instrument can have more than one material, as we can see with the results highlighted. For example, “strumento musicale a percussione” (ending __1296) appears four times in the table, being its materials ‘cuoio’, ‘metallo’, ‘pergamena’, and ‘terracotta’. This should not be a surprise, because in the SELECT- clause we wanted to retrieve all the materials or materials and techniques: in any case, it was an interesting point to examine.

Counting only the instruments associated with one of the properties, we had to eliminate the OPTIONAL keyword, and we also removed from the SELECT part the variables ?Material and ?MaterialOrTechnique, leaving just ?cp. Thus, we ran the following query:

The following SPARQL query retrieves all relevant entries from the ArCo dataset, specifically targeting percussion musical instruments according to their Italian labels.     As we explored the website, we delved into various musical instruments, carefully examining their descriptions and images. We opened numerous records to understand the construction and other properties associated with each instrument.

    During our exploration, we came across a percussion instrument (Ananda Lahari) that intrigued us due to its unique feature: a knob attached to the main body with a string. This detail, visible in the image, was surprising because the instrument's description did not mention a string or its number of components.

    We used the following query to obtain a list of all percussion instruments with their corresponding number of components (if specified):

    PREFIX rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns

    PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#

    PREFIX arco: https://w3id.org/arco/ontology/arco/

    SELECT DISTINCT ?cp ?label ?numberofComponents

    WHERE

    {?cp rdfs:label ?label ;

    rdf:type arco:CulturalProperty .

    OPTIONAL {?cp arco:numberOfComponents ?numberofComponents }

    FILTER(REGEX(?label , "strumento musicale a percussione" , "i"))

    FILTER (lang(?label) = "it")

    }

    ORDER BY (?label)

We asked Gemini and ChatGPT to provide information about their understanding of the specific type of instrument we were interested in, using the Chain-of-Thought prompting technique.

    NEW TRIPLES

    Considering all the previously mentioned information, what we would like to propose is creating two new triples.

    1. The first should specify the fact that the Ananda Lahari is not simply a ‘strumento a percussione’ as described in ArCo, but it is a chordophone which includes some characteristics of a drum. It could be defined as a hybrid instrument incorporating both features of chordophones and percussions, but its primary category should be ‘cordofono’. However, this is probably a detail that only an expert could ascertain, so what we can propose is just a suggestion to change or modify its property with the predicate a-dd:hasCulturalPropertyType.

      2. The corresponding triple could be the following one:
